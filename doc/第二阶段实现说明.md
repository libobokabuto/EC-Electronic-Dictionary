# 第二阶段实现说明

## 已完成内容

### 1. 本地优先 + 在线兜底
- 保留本地词库作为离线基础能力。
- 当本地词库查不到时，程序会自动尝试在线查询。
- 英文词条优先补查在线英语词典，并补充在线翻译结果。
- 中文关键词查不到时，会先翻译成英文候选词，再尝试在线词典补查。

### 2. 统一查询结果展示
- 本地结果与在线结果共用同一套展示界面。
- 新增“来源”字段，明确区分本地词库、在线词典、在线翻译。
- 状态栏会提示当前是本地命中、在线补查成功，还是在线查询失败。

### 3. 本地词库整理
- 替换了原来存在乱码的词库内容。
- 补充了一批更适合第二阶段演示的高频词。
- 支持继续离线演示常见英文词与中文关键词。

### 4. 桌面程序入口
- 新增双击启动脚本：
  `E:\Study\codes\English-Chinese Electronic Dictionary\bin\启动英汉电子词典.bat`
- 新增 Windows 打包脚本：
  `E:\Study\codes\English-Chinese Electronic Dictionary\code\scripts\package-windows.ps1`

## 运行方式

### 方式一：直接双击启动
- 双击 `bin\启动英汉电子词典.bat`
- 如果本地还没有构建产物，脚本会先尝试执行 Maven 打包
- 如果启动失败，窗口现在会停留并显示错误信息，不会再直接闪退

### 方式二：命令行运行
在 `E:\Study\codes\English-Chinese Electronic Dictionary\code` 目录执行：

```powershell
mvn -Dmaven.repo.local=.m2 javafx:run
```

## `mvn package` 产物位置

执行：

```powershell
mvn "-Dmaven.repo.local=.m2" package
```

后可以在下面位置看到程序产物：

```text
E:\Study\codes\English-Chinese Electronic Dictionary\code\target\english-chinese-dictionary-1.0.0.jar
E:\Study\codes\English-Chinese Electronic Dictionary\code\target\lib
```

说明：
- 这一步生成的是 `jar` 和运行依赖，不是最终的 Windows 图形化安装包。
- 如果你想看到更像普通软件的应用目录或 `.exe`，需要继续执行下面的 `jpackage` 打包步骤。

## Windows 打包方式

在 `E:\Study\codes\English-Chinese Electronic Dictionary\code` 目录执行：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\package-windows.ps1
```

打包成功后，输出会出现在：

```text
E:\Study\codes\English-Chinese Electronic Dictionary\code\target\dist
```

## 图标说明

当前打包脚本已经预留图标路径：

```text
E:\Study\codes\English-Chinese Electronic Dictionary\code\assets\app-icon.ico
```

如果后续补上这个 `.ico` 文件，重新执行打包脚本即可自动带入应用图标。

## 当前限制

- 在线查询依赖外部接口，断网时会退回到本地词库能力。
- 免费在线接口返回内容可能不如商业词典稳定。
- 当前已提供 Windows 启动脚本与 `jpackage` 打包脚本，但最终生成 `exe` 仍依赖本机安装 JDK 21。
