# AI 使用记录

## 1. 使用目的
本项目使用 AI 的主要目标是提高课程作业的分析、设计、编码与文档整理效率，同时保留人工判断，确保结果符合课程要求。

## 2. 使用的 AI 能力
- 需求拆解：将老师截图中的作业要求整理为可执行任务清单
- 技术方案建议：结合课程作业场景选择 `JavaFX + Maven + 本地词库`
- 界面设计辅助：生成具有展示效果的布局结构与样式方向
- 代码生成辅助：生成词典实体类、查询服务、持久化服务、JavaFX 主界面
- 测试思路辅助：补充搜索与查无结果场景
- 文档整理：帮助形成任务分析、使用说明与 AI 使用记录

## 3. 关键过程摘要
### 3.1 需求阶段
- 先从老师布置截图提取有效信息
- 明确提交结构必须包含 `code`、`doc`、`bin`
- 明确 `doc` 需要记录 AI 选择、使用过程、对话内容、问题与解决方法

### 3.2 设计阶段
- 将软件拆分为界面层、业务层、数据层
- 决定采用本地 JSON 词库存储词条
- 决定历史与收藏也使用 JSON 落盘，降低工程复杂度

### 3.3 实现阶段
- 生成基础 Maven 工程
- 定义词条数据结构
- 实现搜索、模糊匹配、历史记录、收藏逻辑
- 完成 JavaFX 界面和 CSS 样式
- 编写基础测试与运行说明

## 4. 示例 AI 对话内容摘要
以下是本次开发中可记录到文档里的典型 AI 交互内容摘要：

- “请先阅读截图中的作业要求，并整理成任务文档。”
- “请按 JavaFX 桌面版方向规划英汉电子词典软件。”
- “请基于任务文档直接实现项目，包括 code、doc、bin 结构。”
- “请保证软件支持中英双向检索、模糊联想、历史、收藏和离线运行。”

说明：正式提交时，如果老师明确要求“具体 AI 对话内容”，可以将本次对话的关键截图或文字摘录继续补充到该文档中。

## 5. 人工决策与 AI 辅助边界
- 技术路线选择由人工最终确认
- AI 负责辅助生成方案和代码草稿
- 功能取舍、文档组织、最终测试与验收由人工把关

## 6. 使用 AI 的收益
- 缩短需求分析与工程搭建时间
- 提高文档整理效率
- 帮助形成更完整的界面和功能结构
- 降低从零开始搭建桌面应用的阻力
