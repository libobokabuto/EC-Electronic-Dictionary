# 英汉电子词典软件任务分析与开发说明

## 1. 作业背景
本项目根据课程作业要求，借助 AI 设计并实现一个“英汉电子词典软件”。老师要求提交一个压缩包，压缩包内含 `code`、`doc`、`bin` 三个目录，其中：

- `code`：完整项目源码、资源文件、构建脚本
- `doc`：需求分析、实现说明、AI 使用记录、问题解决过程
- `bin`：可直接运行的程序及运行说明

本实现选择 `JavaFX + Maven + 本地 JSON 词库` 技术路线，目标是在满足课程要求的前提下，呈现较好的桌面界面效果与可演示性。

## 2. 需求分析
软件必须覆盖以下核心能力：

- 英文查中文
- 中文查英文
- 支持模糊搜索与联想提示
- 显示词性、释义、例句、备注
- 保存最近查询历史
- 收藏常用词条
- 提供空输入、查无结果、重复收藏等异常反馈

为了提升展示效果，本项目额外加强了：

- 具有视觉层次的首页式头部区域
- 侧边栏历史与收藏结构
- 明确的状态栏反馈
- 本地离线可运行，便于课堂演示

## 3. 系统设计
### 3.1 分层结构
- 表现层：JavaFX 页面、事件处理、结果渲染、样式文件
- 业务层：词条查询、联想提示、历史管理、收藏管理
- 数据层：本地 JSON 词库、历史记录文件、收藏记录文件

### 3.2 数据流
1. 用户在搜索框输入英文单词或中文关键词。
2. 程序先进行标准化处理，再交给查询服务匹配词库。
3. 若找到结果，则渲染词条详情，并记录历史。
4. 若点击收藏，则写入本地收藏文件。
5. 历史和收藏在软件下次启动时自动读取。

### 3.3 主要目录结构
```text
code/
  pom.xml
  src/main/java/com/ecdictionary/...
  src/main/resources/data/dictionary.json
  src/main/resources/styles/app.css
doc/
  任务分析与开发说明.md
  AI使用记录.md
  使用说明.md
bin/
  run.bat
  README.txt
```

## 4. 技术选型说明
### 4.1 为什么选择 JavaFX
- 更适合桌面 GUI 作业展示
- 相比 Swing 更现代，布局和样式更灵活
- 支持 CSS，可较方便地优化视觉效果

### 4.2 为什么选择 Maven
- 便于管理 JavaFX 与 JSON 依赖
- 便于执行测试、打包、后续维护

### 4.3 为什么选择本地 JSON 词库
- 简单直观，适合课程作业
- 无需联网，现场演示稳定
- 词条结构清晰，便于扩展和修改

## 5. 已实现功能

- 中文关键词查询
- 模糊联想推荐

- 最近历史记录
- 收藏与取消收藏
- 本地持久化
- 单元测试覆盖关键检索逻辑
