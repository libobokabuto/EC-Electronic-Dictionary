# 第二阶段任务规划

## 1. 第二阶段目标
第二阶段的目标，是将当前“课程演示版词典”升级为“更接近真实软件的可用版词典”，重点解决两个最明显的问题：

- 很多单词查不出来
- 启动方式还不像普通桌面应用

## 2. 你明确提出的两个方向
### 方向一：实现联网能力
目标：

- 当本地词库查不到单词时，程序可以联网查询
- 提高查词覆盖面
- 让软件更接近真实词典 App

建议实现方式：

- 保留本地词库，继续作为离线基础能力
- 新增在线查询服务
- 查询策略改为：
  - 先查本地
  - 本地无结果再请求在线接口
  - 在线成功后展示返回结果
  - 在线失败时给出明确提示

预期收益：

- 绝大多数常见单词都可查询
- 支持更真实的使用体验
- 演示时更容易体现软件完整性

### 方向二：做成像普通 App 一样的桌面程序
目标：

- 双击图标直接打开
- 不依赖手动运行命令
- 更像正式软件，而不是开发项目

建议实现方式：

- 基于当前 JavaFX 工程进行 Windows 桌面打包
- 生成 `exe` 或原生应用目录
- 配置应用图标、名称、启动入口
- 尽量让用户无需关心 Java 命令与依赖细节

预期收益：

- 演示更自然
- 提交给老师时观感更好
- 更符合“软件作品”而不是“源码作业”

## 3. 我建议追加的第二阶段功能
除了你提出的两个主方向，我建议第二阶段一并考虑以下增强项。

### 3.1 扩充本地词库
即使有联网能力，也建议保留并扩充本地词库。

原因：

- 断网时仍可使用
- 常用词可更快返回
- 便于演示“离线 + 在线”的双模式能力

### 3.2 统一查询结果展示结构
联网后返回的数据结构可能和本地词库不同，因此第二阶段应统一结果模型，确保：

- 本地结果和在线结果共用同一套展示界面
- 词性、释义、例句、来源字段显示一致
- UI 不因数据来源不同而混乱

### 3.3 错误提示与网络状态提示
新增联网后，必须补强以下提示：

- 网络不可用
- 接口超时
- 查询失败
- 返回结果为空
- 当前结果来自本地还是在线

### 3.4 应用图标与品牌信息
如果做 `exe`，建议顺手完善：

- 应用图标
- 应用名称
- 窗口标题
- 简要版权或课程项目信息

### 3.5 更完整的文档补充
第二阶段完成后，应补充：

- 在线接口接入说明
- 打包说明
- 安装与运行说明
- 新增问题及解决记录

## 4. 第二阶段建议实施顺序
建议按下面顺序推进：

1. 扩充并整理本地词库
2. 设计统一查询结果模型
3. 接入联网查询接口
4. 完善在线失败与状态提示
5. 联调 UI 展示效果
6. 打包为 Windows 桌面应用
7. 增加图标与启动入口优化
8. 更新第二阶段文档

## 5. 第二阶段验收标准
第二阶段完成后，建议至少达到以下结果：

- 输入常见英文单词，绝大部分可以查到
- 输入中文关键词，返回更合理的结果
- 离线时本地词库仍可工作
- 联网时查询覆盖明显增强
- 双击应用图标即可启动
- 不需要用户输入启动命令
- 文档中能清楚说明第二阶段新增内容

## 6. 第二阶段定位
第二阶段完成后，项目应从：

**课程演示版词典**

升级为：

**具备联网查词与桌面应用形态的增强版英汉电子词典软件**
