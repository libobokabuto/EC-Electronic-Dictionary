# 第三阶段任务规划

## 1. 第三阶段定位

第三阶段不是继续大幅扩展新能力，而是对第二阶段成果进行：

- 修瑕疵
- 做收尾
- 做提交物整理

目标是把当前项目从“已经实现主要功能的增强版作品”打磨成“可以直接展示、直接压缩、直接提交”的最终版本。

## 2. 第三阶段总体目标

第三阶段建议围绕以下四个方向推进：

### 方向一：修复现有瑕疵

重点处理第二阶段遗留的小问题和不稳定点，例如：

- 打包后的 `exe` 或应用目录启动异常
- 启动配置中的 JavaFX 模块或缓存路径问题
- 在线查询流程中的边界提示不够清晰
- 词条展示中的少量文案、格式或体验问题
- 目录结构与文件命名不够统一的问题

### 方向二：完成最终打包

目标：

- 确保最终可生成可运行的 Windows 桌面版产物
- 明确最终交付到底采用 `app-image`、`exe` 还是两者同时保留
- 确保双击可以正常启动

建议完成内容：

- 修通 `jpackage` 最终配置
- 统一应用名、窗口名、启动器名称
- 如条件允许，加入正式图标
- 保证最终产物目录清晰

### 方向三：压缩与提交物整理

目标：

- 生成适合提交给老师或课程展示的最终压缩包

建议完成内容：

- 整理最终目录结构
- 去掉不必要的中间调试文件
- 明确哪些文件需要打包进提交物
- 生成最终压缩包

### 方向四：补全 README 与收尾文档

目标：

- 让任何看到项目的人都能快速理解怎么运行、怎么打包、项目做了什么

建议完成内容：

- 补全 README
- 补充最终运行方式说明
- 补充打包方式说明
- 补充项目亮点总结
- 补充已知问题与后续可改进方向

## 3. 第三阶段建议任务拆分

### 3.1 启动与打包问题修复

建议优先检查：

- `EnglishChineseDictionary.exe` 是否可正常启动
- `app-image` 内启动器配置是否正确
- JavaFX 模块路径是否被正确传入
- 缓存目录是否写入到可用位置
- 打包脚本在不同参数下是否都能运行

验收目标：

- 双击可启动
- 命令行可启动
- 打包脚本可稳定复用

### 3.2 体验细节修复

建议处理：

- 查询失败提示文案优化
- 在线结果与本地结果展示一致性再检查
- 收藏、历史、联想功能与第二阶段新逻辑的兼容性复查
- UI 中个别文字、间距、默认提示是否足够自然

验收目标：

- 演示时不容易出现突兀、模糊、混乱的提示

### 3.3 资源与品牌信息整理

建议处理：

- 补充正式应用图标
- 统一应用名称
- 统一窗口标题
- 补充简短版权或课程项目信息

验收目标：

- 更像完整软件作品，而不是临时演示工程

### 3.4 README 与文档补全

README 建议至少包含：

- 项目简介
- 功能特性
- 运行方式
- 打包方式
- 目录说明
- 第二阶段与第三阶段成果说明
- 已知问题

验收目标：

- 不看聊天记录，只看 README 也能理解项目

### 3.5 最终压缩与提交物生成

建议准备两类交付物：

- 源码版提交物
- 可运行桌面版提交物

建议压缩包中至少明确包含：

- `code`
- `doc`
- 必要的 `bin`
- 最终桌面应用目录或安装包
- README

验收目标：

- 教师或同学拿到压缩包后能顺利查看和运行

## 4. 第三阶段建议实施顺序

建议按以下顺序推进：

1. 修复桌面启动与打包问题
2. 联调应用运行体验
3. 整理图标、名称、窗口标题等细节
4. 补全 README 和最终说明文档
5. 清理中间文件
6. 生成最终压缩包

## 5. 第三阶段验收标准

第三阶段完成后，建议至少达到以下结果：

- 双击桌面版程序可以正常启动
- 打包脚本能够稳定生成最终产物
- 项目目录清晰，不再混乱
- README 内容完整，能独立说明项目
- 提交压缩包结构清楚
- 项目可以作为课程作品直接提交或展示

## 6. 第三阶段输出物建议

第三阶段完成后，建议最终输出：

- 最终源码目录
- 最终桌面应用目录或安装包
- 完整 README
- 最终说明文档
- 最终压缩包

## 7. 第三阶段工作原则

这一阶段建议遵循三个原则：

- 不再大规模扩展新功能，避免引入新风险
- 优先解决已知问题，把现有成果真正跑通
- 所有工作以“适合展示、适合提交、适合归档”为标准

## 8. 第三阶段与第二阶段的关系

第二阶段解决的是“能力增强”问题：

- 能联网
- 能统一展示
- 更像桌面软件

第三阶段解决的是“最终交付”问题：

- 是否真正稳定
- 是否能直接运行
- 是否能直接提交
- 是否有完整文档

因此第三阶段可以理解为：

**把第二阶段已经做出来的增强版词典，打磨成最终可交付版本。**
